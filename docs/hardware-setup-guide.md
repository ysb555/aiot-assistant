## 2. 硬件接线

### 2.1 I2C0 总线设备 (PB8=SCL, PB9=SDA)

将所有I2C0设备的 SCL 和 SDA 分别连接到主控板的对应引脚。
需要连接 VCC(3.3V) 和 GND。

```
主控板(U1P)              外设板
┌──────────┐         ┌──────────┐
│ PB8(SCL) ├─────────┤  SCL     │
│ PB9(SDA) ├─────────┤  SDA     │
│ 3.3V     ├─────────┤  VCC     │
│ GND      ├─────────┤  GND     │
└──────────┘         └──────────┘
```

**I2C0 上拉电阻**：SCL 和 SDA 各接 4.7kΩ 上拉到 3.3V。

### 2.2 S2 传感器板（温度/湿度/光照）

| 传感器 | I2C地址 | 总线 | 量程 | 精度 |
|--------|---------|------|------|------|
| SHT35  | 0x44    | I2C0 | -40~125°C / 0~100%RH | ±0.1°C / ±1.5%RH |
| BH1750 | 0x23    | I2C0 | 1~65535 lux | ±20% |

- SHT35 ADDR引脚接GND → 地址=0x44；接VCC → 地址=0x45
- BH1750 ADDR引脚接GND → 地址=0x23；接VCC → 地址=0x5C

### 2.3 S1 按键板（4键矩阵按键）
- 地址：0x24（I2C0 总线）
- 按键功能定义：
  - 按键1 (0x01)：切换显示模式 (评分→温度→湿度→光照)
  - 按键2 (0x02)：切换植物档案 (绿萝→多肉→发财树)
  - 按键3 (0x04)：手动开关自动遮阳
  - 按键4 (0x08)：手动开关自动通风

### 2.4 E1 输出板（RGB灯 + 数码管）

| 芯片 | I2C地址 | 功能 | 通道 |
|------|---------|------|------|
| PCA9685 | 0x40 | PWM输出 | CH0=绿灯, CH1=红灯, CH2=蓝灯 |
| HT16K33 | 0x70 | 4位数码管 | 显示评分/温度/湿度/光照 |

RGB灯颜色映射：
- 绿(0,255,0)：综合评分90-100 环境极佳
- 黄(255,255,0)：综合评分70-89
- 橙(255,165,0)：综合评分50-69
- 红(255,0,0)：综合评分<50 环境堪忧

### 2.5 E2 风扇板（PWM风扇控制）
- PCA9685 地址 0x60，通过PWM驱动直流风扇
- 风扇转速：
  - 低速：PWM 50%占空比
  - 高速：PWM 90%占空比

### 2.6 C3 WiFi模块（ESP32-C3）
- 连接方式：UART0 (PA9=TX, PA10=RX)，波特率115200
- 协议：AT 指令集
- 上电后自动连接指定 WiFi，通过 TCP 发送浇水提醒 HTTP GET 请求
- 需修改 `c3.h` 中的 WiFi 和服务器配置：
  ```c
  #define WIFI_SSID       "YourWiFiSSID"
  #define WIFI_PASSWORD   "YourPassword"
  #define SERVER_IP       "192.168.1.100"
  #define SERVER_PORT     "8080"
  #define WATER_ALERT_PATH "/api/water-alert"
  ```

### 2.7 U2P 视觉模块
- 连接方式：UART2 (PC10=TX, PC11=RX)，波特率115200
- 协议：定期发送识别结果，格式 `NAME:xxx SCORE:0.x`
- 通过 u1p_fusion_decision() 融合视觉结果到控制决策中

## 3. 供电方案

| 模块 | 供电电压 | 典型电流 | 说明 |
|------|----------|----------|------|
| GD32F470 主控 | 3.3V | 100mA | 由USB或外部5V通过LDO降压 |
| S2 传感器 | 3.3V | 10mA | 由主控板供电 |
| S1 按键 | 3.3V | 5mA | 由主控板供电 |
| E1 RGB+数码管 | 5V | 200mA | RGB LED 电流较大，建议独立5V |
| E2 风扇 | 5V/12V | 200-500mA | 视风扇规格，建议独立供电 |
| C3 WiFi | 3.3V | 150mA | 峰值发射电流 |
| U2P 视觉 | 3.3V/5V | 视具体模块 | - |

**推荐供电**：
- 使用 5V/2A 适配器给主控板
- 风扇和RGB灯如电流大，使用独立12V电源 + DC-DC降压模块

## 4. 搭建步骤

### 第一步：准备材料
- [ ] GD32F470 开发板 ×1
- [ ] SHT35 温湿度传感器模块 ×1
- [ ] BH1750 光照传感器模块 ×1
- [ ] PCA9685 PWM驱动模块 ×2（一个给RGB，一个给风扇）
- [ ] HT16K33 4位数码管模块 ×1
- [ ] RGB LED 共阴极 灯环或灯珠 ×1
- [ ] ESP32-C3 AT固件模块 ×1
- [ ] 4键矩阵按键模块 ×1
- [ ] 5V/12V 直流风扇 ×1（PWM调速）
- [ ] I2C 连接线（杜邦线母对母）
- [ ] 4.7kΩ 电阻 ×4（I2C上拉）
- [ ] 5V/2A 电源适配器 ×1
- [ ] 面包板或PCB底板 ×1
- [ ] 遮阳帘+舵机/步进电机（可选，用于自动遮阳）

### 第二步：焊接/连接 I2C 总线
1. 将所有 I2C0 设备的 SCL 线并接到 PB8
2. 将所有 I2C0 设备的 SDA 线并接到 PB9
3. 在总线上加上 4.7kΩ 上拉电阻到 3.3V
4. 连接 VCC(3.3V) 和 GND 到各模块

### 第三步：连接 UART 设备
1. C3 WiFi：PA9(TX)→C3 RX，PA10(RX)→C3 TX
2. U2P 视觉：PC10(TX)→U2P RX，PC11(RX)→U2P TX

### 第四步：烧录固件
1. 修改 `c3.h` 中的 WiFi SSID、密码和服务器地址
2. 使用 GCC ARM Toolchain 编译：
   ```
   cd demo/Project/CMAKE_Project
   mkdir build && cd build
   cmake .. -G "Ninja"
   ninja
   ```
3. 烧录到 GD32F470：
   ```
   openocd -f interface/stlink.cfg -f target/stm32f4x.cfg -c "program build/PLANT_ASSISTANT.hex verify reset exit"
   ```

### 第五步：上电测试
1. 检查数码管显示（应显示植物编号）
2. RGB灯应随评分变色
3. 按键操作切换显示模式和植物档案
4. 通过串口调试助手检查传感器数据 (`TEMP?`/`HUMI?`/`LIGHT?`/`SCORE?`)

## 5. 软件架构说明

### 5.1 调度方案（无RTOS轮询）
```
SysTick (1ms)
  └─ 每50ms
      ├─ 按键扫描
      ├─ UART命令解析
      └─ 每1秒 (20×50ms)
          ├─ 读取传感器
          ├─ 计算评分
          ├─ 蒸发指数/浇水提醒
          ├─ 自动遮阳 (回差控制)
          ├─ 自动通风 (回差控制)
          ├─ WiFi连接状态机
          └─ 更新显示/RGB
```

### 5.2 控制算法
- **综合评分** = 温度×0.3 + 湿度×0.4 + 光照×0.3
- **蒸发指数** = 温度×0.3 + 光照×0.5 - 湿度×0.2
- **遮阳回差控制**：光照 > 高阈值 → 关 / < 低阈值 → 开（避免频繁切换）
- **通风回差控制**：湿度 > 高阈值 → 开扇 / < 低阈值 → 关扇

### 5.3 预置植物档案
| 植物 | 温度范围 | 湿度范围 | 光照范围 | 喜光性 |
|------|---------|---------|---------|--------|
| 绿萝 | 18-28°C | 50-70% | 200-500 lux | 喜阴湿 |
| 多肉 | 15-25°C | 30-50% | 500-1000 lux | 喜光耐旱 |
| 发财树 | 20-30°C | 40-60% | 300-600 lux | 温暖湿润 |

## 6. 故障排查

| 现象 | 可能原因 | 解决办法 |
|------|---------|---------|
| 数码管不亮 | HT16K33未初始化 | 检查I2C0地址0x70，确认4.7kΩ上拉电阻 |
| RGB不亮 | PCA9685地址冲突 | 检查地址跳线，确认0x40未与风扇板冲突 |
| 传感器读数为0 | SHT35/BH1750未上电或地址错误 | 逐一检查I2C地址扫描结果 |
| 风扇不转 | 供电不足 | 换用独立12V供电 |
| WiFi连不上 | SSID/密码错误或信号弱 | 串口监控AT指令返回，检查C3固件 |
| 按键无反应 | I2C通讯失败 | 确认S1板地址0x24，检查上拉电阻 |
