
## AIoT 实训箱硬件系统全面说明

你好！这是一套用于教学和开发的 **AIoT（人工智能+物联网）实训箱**。为了让你（AI）能准确理解这套硬件平台的工作原理、搭建方式和用途，下面我将从一个“完全不了解具体板卡”的角度，用类比和结构化描述来总结所有资料内容。

### 一、整体思想：模块化堆叠式硬件平台

这套系统采用 **“母板 + 子板”** 的模块化设计，类似于搭建积木：

- **母板（M1）**：是基础底座，提供电源、总线和物理连接接口。母板水平放置，上面有多个插槽（如 #0~#5）。
- **子板**：是功能模块，通过磁吸脚或堆叠方式固定在母板上，通过 **Pogo Pin（弹簧针）** 与母板通信。子板可以随时更换组合。
- **计算子板（U系列）**：是整个系统的“大脑”，运行主控程序。有 U1（GD32F470 单片机）和 U2（地平线旭日 X3M 高性能 AI 芯片）等。
- **传感器子板（S系列）**：采集环境信息，如温度、湿度、光照、声音、人体红外、NFC、按键等。**绿色板**。
- **执行器子板（E系列）**：执行动作，如点亮 LED 数码管、转动风扇、开关窗帘、播放声音、显示屏幕等。**黄色板**。
- **通信子板（C系列）**：提供无线通信能力，如 ZigBee、Wi-Fi + 蓝牙。
- **扩展子板（B系列）**：扩展外部传感器接口（如霍尔传感器、气压传感器）。

> **关键点**：所有子板通过 **I2C 总线**（两根线：SDA 数据线、SCL 时钟线）与计算子板通信。I2C 是一种低成本、多设备的串行通信协议，每个子板都有一个可配置的 I2C 地址（通过拨码开关设置），计算子板通过地址来访问特定子板。

### 二、硬件组成清单与功能

#### 1. 计算子板（主控）

| 型号 | 核心芯片 | 资源 | 开发方式 |
|------|----------|------|----------|
| **U1 / U1P** | GD32F470 (ARM Cortex-M4) | 3MB Flash, 768KB RAM, 240MHz | C/C++，用 Keil/IAR/VSCode，编译后烧录 |
| **U2 / U2P** | 地平线旭日 X3M | 4核 ARM A53, 5 TOPS AI算力, 4GB RAM, 16GB eMMC | Linux 系统，串口或 SSH 登录，运行 C/Python，支持 AI 模型（YOLO等） |

- **U1** 适合实时控制、读取传感器、驱动执行器。
- **U2** 适合跑 Linux、做 AI 推理（图像识别、语音处理），算力强，功耗低。

#### 2. 传感器子板（S系列）—— 采集数据

| 型号 | 功能 | 关键芯片 / 原理 | I2C地址示例 | 输出数据 |
|------|------|----------------|--------------|----------|
| S1 | 3×4 矩阵按键 | HT16K33 | 0xE8/EA/EC/EE | 按键值 |
| S2 | 光照强度 | BH1750 | 0x46 / 0xB8 | 1~65535 lux |
| S3 | 麦克风（模拟+数字阵列） | MEMS麦克风 + PDM | 无直接I2C | 音频数据（需 ALSA） |
| S4 | 摄像头（图片中提及） | 未详细说明 | - | 图像数据 |
| S5 | NFC读卡 | MS523 | 0x50~0x56 | 卡片UID |
| S6 | 超声波测距 | DYP-A06 | 0x58~0x5E | 距离(cm) |
| S7 | 人体红外（热释电） | BS412 + PCA9557 | 0x30~0x36 | 有人/无人（1/0） |
| S8 | 温湿度 | SHT35 | 0x88 / 0x8A | 温度（-40~125℃），湿度（0~100%RH） |

#### 3. 执行器子板（E系列）—— 执行动作

| 型号 | 功能 | 关键芯片 | I2C地址 | 控制方式 |
|------|------|----------|---------|----------|
| E1 | RGB三色灯 + 4位数码管 | PCA9685 (PWM调光) + HT16K33 (数码管) | 灯: 0xC0~0xC6<br>数码管: 0xE0~0xE6 | 写入亮度寄存器 / 显示数字 |
| E2 | 直流风扇（调速） | PCA9685 | 0xC8~0xCE | PWM占空比控制转速 |
| E3 | 窗帘电机（直流电机+限位） | HR8833 + 辅助MCU | 0x38~0x3E | 写入位置0~100% |
| E4 | 扬声器（音频播放） | NS4160 功放 | 通过 I2S 总线 | 音频数据流（非I2C） |
| E5 | 8寸触摸屏 | HDMI + USB触控 | 无I2C | 显示 + 触摸坐标 |

| 型号 | 通信方式 | 模组 | 作用 | 控制接口 |
|------|----------|------|------|-----------|
| C2 | ZigBee | E18-MS1-PCB (CC2530) | 低功耗自组网 | 串口 AT 指令 |

### 三、系统搭建与工作方式

#### 物理搭建
1. 将 **母板** 平放，接上电源（5V）。
2. 选择需要的 **计算子板**（U1 或 U2），放在母板指定位置。
3. 根据需要，将传感器子板（S）、执行器子板（E）、通信子板（C）堆叠在母板其他槽位或堆叠在U板上面。注意某些子板有堆叠顺序要求（如 E4 必须放在 U 板之上）。
4. 所有子板通过母板内部的总线（I2C、电源、地）自动连接。

#### 通信方式
- **主要总线**：I2C。每个子板上有一个 **拨码开关**，用于设置 I2C 地址，避免地址冲突。计算子板通过 `I2C_write` / `I2C_read` 与子板交换数据。
- **其他接口**：UART（用于 C2/C3 无线模块）、I2S（用于 E4 音频）、HDMI+USB（用于 E5 屏幕）、以太网（U2 调试）。

#### 软件开发层次
1. **硬件层**：具体芯片寄存器。
2. **标准外设库**：厂商提供的 API，直接操作 GD32 的 I2C、GPIO 等。
3. **BSP（板级支持包）**：针对每个子板封装好的高级 API，例如 `e1_led_set_color()`、`e3_curtain_move_to(50)`。**开发应用时优先使用 BSP**。
4. **应用层**：用户编写的业务逻辑，如智能门锁、环境监测等。

### 四、典型应用示例（帮助理解整体）

#### 示例1：点亮 E1 板上的 RGB 灯并显示数字
- **硬件**：U1 + E1
- **流程**：
  - 初始化 I2C 总线。
  - 向 PCA9685（灯驱动芯片）写入寄存器，设置红、绿、蓝三路的 PWM 占空比，混合出想要的颜色。
  - 向 HT16K33（数码管驱动）写入命令，设置要显示的数字（如 `3.14`）。
- **代码**（伪）：
  ```c
  e1_led_set_rgb(255, 0, 0);  // 红色
  e1_dig_set_hex_num(1234);   // 显示 1234
  ```

#### 示例2：读取温湿度并控制风扇
- **硬件**：U1 + S8（温湿度） + E2（风扇）
- **流程**：
  - 通过 I2C 读取 S8 的温湿度值（计算得到实际温度、湿度）。
  - 如果温度 > 30℃，则通过 I2C 向 E2 的 PCA9685 写入高占空比，风扇快速转动。
- **关键**：所有子板共享同一条 I2C 总线，U1 作为主机，分别访问不同地址的设备。

#### 示例3：U2 做 AI 图像识别控制 E3 窗帘
- **硬件**：U2 + S4（摄像头） + E3（窗帘电机）
- **流程**：
  - U2 上运行 Linux，用 Python 调用摄像头，跑 YOLO 模型检测手势。
  - 识别到“向左挥手” → 通过 I2C 向 E3 发送命令（写入 0~100 的位置值），电机转动打开窗帘。
- **AI 模型部署**：U2 支持将训练好的浮点模型量化为 INT8 模型，用地平线工具链转换后运行。

#### 示例4：智能门锁综合应用（资料中给出的复杂示例）
- **用到的子板**：U1, U2, S1(按键), S5(NFC), E1(数码管), E3(窗帘), E4(扬声器), E5(屏幕), S3(麦克风), S4(摄像头), C2(ZigBee) ×2
- **功能推测**：
  - 通过 S1 输入密码，S5 刷 NFC 卡，S4 人脸识别（AI）。
  - 验证成功 → 打开电磁锁（未列出的执行器），E1 显示成功，E4 播放语音，E5 屏幕显示欢迎。
  - 通过 C2 ZigBee 与另一个 ZigBee 节点通信，实现远程门禁。
- **管理模式**：长按 12 号键进入，可删除密码、NFC、面容等。

### 五、开发环境与调试方法

#### 针对 U1（单片机）
- **IDE**：VSCode + C/C++ 插件 + Cortex-Debug。
- **工具链**：arm-none-eabi-gcc, cmake, ninja, openocd。
- **编译烧录**：运行 `build_and_flash.bat`，通过下载器将固件写入 U1。
- **调试**：VSCode 中按 F5，选择对应配置，可单步执行、查看变量。

#### 针对 U2（Linux + AI）
- **连接方式**：
  - 串口（USB 转 TTL）：MobaXterm 等工具，波特率 115200，用户名 root，密码 root。
  - 网络（以太网或 WiFi）：`ssh root@<ip>`，推荐用 VSCode Remote-SSH。
- **编译运行**：
  - C 程序：`make clean && make`，然后 `sudo ./E1-demo`。
  - Python 程序：直接 `python3 xxx.py`。
- **AI 模型**：可从地平线官方 Boxs 仓库获取预训练模型（YOLO、FCOS、手势识别等），也可自己训练后转换。

### 六、关键术语解释

| 术语 | 解释 |
|------|------|
| **I2C** | 两线制串行总线，用于低速设备间通信。每个设备有唯一地址。 |
| **PWM** | 脉宽调制，通过改变方波占空比来模拟模拟量（如亮度、转速）。 |
| **BSP** | 板级支持包，对硬件功能的封装，让开发者不用关心寄存器细节。 |
| **TOPS** | 每秒万亿次运算（INT8），衡量 AI 算力的单位。U2 为 5 TOPS。 |
| **拨码开关** | 子板上的小开关，用于设置 I2C 地址或功能选择。 |
| **堆叠** | 将子板通过磁吸或插针叠放在母板或另一块子板上。 |
| **Pogo Pin** | 弹簧顶针，用于板间可靠连接。 |

### 七、总结（给 AI 的理解要点）

> 这是一套 **模块化、可组合** 的 AIoT 教学平台。核心是一块**母板**，上面可以**堆叠**不同功能的**子板**：传感器（S，绿色）、执行器（E，黄色）、主控（U，大脑）、无线模块（C）、扩展板（B）。所有子板通过 **I2C 总线** 和主控通信，主控通过读写 I2C 地址来获取数据或控制动作。开发时，底层有标准外设库，上层有封装好的 BSP API，方便快速编写应用。其中 U2 子板具备 **5 TOPS AI 算力**，可运行 Linux 和深度学习模型，实现语音、图像识别。整个系统可以搭建出智能家居、门禁、环境监测、工业控制等典型物联网场景。
