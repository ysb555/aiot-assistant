# 植物管家 - 硬件搭建与运行指南

---

## 一、硬件架构概述

植物管家系统基于 **小米AIoT实训套件**，采用 **"母板 + 子板"** 模块化架构设计。

### 1.1 母板（M1）

母板是整个系统的**基础底座**，提供：
- 电源分配（为所有子板供电）
- I2C 通信总线
- 物理插槽（**槽位 #0 ~ #5**，共 6 个插槽）

> 母板本身不运行任何程序，它只是承载子板的平台。主控程序运行在插入母板的**计算子板**上。

### 1.2 子板分类

| 系列 | 颜色 | 作用 | 代表型号 |
|------|------|------|----------|
| **U 系列（计算子板）** | - | 系统的"大脑"，运行主控程序 | U1 (GD32F470)、U2 (地平线旭日 X3M) |
| **S 系列（传感器子板）** | 绿色 | 采集环境信息 | S1 (按键)、S2 (温湿度+光照+6轴)、S5 (NFC)、S6 (超声波)、S7 (人体红外) |
| **E 系列（执行器子板）** | 黄色 | 执行动作 | E1 (RGB灯+数码管)、E2 (风扇)、E3 (窗帘电机)、E4 (扬声器) |
| **C 系列（通信子板）** | - | 无线通信 | C2 (ZigBee)、C3 (Wi-Fi + 蓝牙) |
| **B 系列（扩展子板）** | - | 外部传感器接口扩展 | 霍尔传感器、气压传感器等 |

## 二、硬件清单与槽位分配

### 2.1 需要的硬件

| 编号 | 子板 | 功能 | 插入槽位 | I2C 地址 (8位) |
|------|------|------|----------|-----------------|
| **U1** | GD32F470 计算子板 | 运行主控程序 | #0 | - |
| **S2** | 三合一传感器板 | SHT35(温湿度) + BH1750(光照) + ICM20608(6轴) | #1 | 0x88 / 0x46 / 0xD0 |
| **S1** | 矩阵按键板 | 4x4 矩阵键盘 | #2 | HT16K33 |
| **S5** | NFC 读卡器 | 读取NFC标签切换植物 | #3 | MS523 |
| **S6** | 超声波测距 | 水位/土壤检测 | #4 | GD32F330 |
| **S7** | 人体红外(PIR) | 检测是否有人 | #5 | PCA9557 |
| **E1** | RGB灯 + 数码管 | 评分颜色指示 + 数值显示 | 扩展槽 | PCA9685(灯) + HT16K33(数码管) |
| **E2** | 风扇模块 | 通风散热 | 扩展槽 | PCA9685 |
| **E3** | 窗帘电机 | 自动遮阳 | 扩展槽 | 步进电机驱动 |
| **E4** | 扬声器模块 | 蜂鸣提示 | 扩展槽 | WM8978 |
| **C2** | ZigBee 模块 | 无线数据广播/接收 | 通信接口 | UART2 (PC10/PC11) |
| **C3** | WiFi + 蓝牙模块 | WiFi联网 + HTTP请求 | 通信接口 | UART0 (PA9/PA10) |

### 2.2 S2 三合一传感器板详解

> **重要：** S2 是**一块物理子板**上集成了三个传感器芯片，**不存在** S8 这样的独立温湿度子板。

| 传感器芯片 | 功能 | I2C 地址 (8位) | 拨码开关选项 |
|------------|------|----------------|--------------|
| **SHT35** | 温度 + 湿度 | **0x88** | 0x88, 0x8A |
| **BH1750** | 光照度 (lux) | **0x46** | 0x46, 0xB8 |
| **ICM20608** | 6轴陀螺仪/加速度计 (可选) | **0xD0** | 0xD0, 0xD2 |

> 拨码开关用于在同一个 I2C 总线上接多个同类型子板时避免地址冲突。本项目每种只用一块，使用默认地址即可。

## 三、物理搭建步骤

### 步骤 1：准备母板 M1
将母板放置在平稳的工作台上，确认电源接口可用。

### 步骤 2：插入计算子板 U1
将 **U1 (GD32F470 单片机)** 计算子板插入母板的 **#0 槽位**。U1 是整个系统的核心，所有控制程序都运行在 U1 上。
- U1 通过 USB 线连接电脑，用于**烧录程序**和**串口通信**
- U1 上的 PA9/PA10 (UART0) 经 USB 转串口芯片引出到电脑

### 步骤 3：插入传感器子板（S 系列，绿色板）
按槽位依次插入：
1. **S2** → #1 槽位（温湿度 + 光照 + 6轴）
2. **S1** → #2 槽位（矩阵按键）
3. **S5** → #3 槽位（NFC 读卡器）
4. **S6** → #4 槽位（超声波测距）
5. **S7** → #5 槽位（人体红外）

### 步骤 4：连接执行器子板（E 系列，黄色板）
执行器子板通过排线连接到母板的扩展接口：
1. **E1** (RGB灯+数码管) → 扩展接口
2. **E2** (风扇) → 扩展接口
3. **E3** (窗帘电机) → 扩展接口
4. **E4** (扬声器) → 扩展接口

### 步骤 5：连接通信子板（C 系列）
1. **C3 (WiFi+蓝牙)** → 连接到 UART0 引脚 (PA9/PA10)，与电脑共用同一串口
   - 注：UART0 同时用于固件下载、调试输出和 C3 通信，由软件通过 AT 指令模式切换
2. **C2 (ZigBee)** → 连接到 UART2 引脚 (PC10/PC11)

### 步骤 6：连接 U2 视觉子板（可选）
如果有 **U2 (地平线旭日 X3M)** 视觉识别子板：
- 插入另一个槽位
- U2 通过 USART1 (PA2/PA3) 与 U1 通信

### 步骤 7：连接电源
通过母板电源接口给整个系统供电（通常 USB 5V 或 DC 12V）。

### 步骤 8：连接电脑
用 USB 线连接 U1 计算子板到电脑：
- 用于烧录固件（通过 J-Link / DAP-Link 或串口 ISP）
- 用于串口通信（PC 前端控制面板通过此串口发送命令）

## 四、I2C 地址系统说明

所有子板通过 I2C 总线与 U1 通信。I2C 地址使用 **8 位左移地址**（即 7 位地址 << 1）。

| 子板 | 传感器/芯片 | 7位地址 | 8位地址 (代码使用) |
|------|-------------|---------|-------------------|
| S2 | SHT35 (温湿度) | 0x44 | **0x88** (0x88/0x8A) |
| S2 | BH1750 (光照) | 0x23 | **0x46** (0x46/0xB8) |
| S2 | ICM20608 (6轴) | 0x68 | **0xD0** (0xD0/0xD2) |
| S1 | HT16K33 (按键) | - | **get_board_address() 扫描** |
| S5 | MS523 (NFC) | - | **get_board_address() 扫描** |
| S6 | GD32F330 (超声波) | - | **get_board_address() 扫描** |
| S7 | PCA9557 (红外) | - | **get_board_address() 扫描** |
| E1 | PCA9685 (RGB) | - | **get_board_address() 扫描** |
| E1 | HT16K33 (数码管) | - | **get_board_address() 扫描** |
| E2 | PCA9685 (风扇) | - | **get_board_address() 扫描** |

### 拨码开关地址偏移规则
每个子板有拨码开关，可配置 2 位 = 4 种地址：
- 00 → 基础地址 + 0
- 01 → 基础地址 + 2
- 10 → 基础地址 + 4
- 11 → 基础地址 + 6

### 5.2 编译固件

```bash
# 进入构建目录
cd plant_assistant/demo/Project/CMAKE_Project

# 创建 build 目录
mkdir build && cd build

# CMake 配置 (使用 arm-none-eabi-gcc 工具链)
cmake .. -DCMAKE_TOOLCHAIN_FILE=<your_toolchain>.cmake

# 编译
cmake --build .
```

编译产物：
- `PLANT_ASSISTANT.elf` - ELF 调试文件
- `PLANT_ASSISTANT.hex` - HEX 烧录文件
- `PLANT_ASSISTANT.bin` - BIN 烧录文件

### 5.3 烧录到 U1

使用 J-Link 或 DAP-Link 将 `PLANT_ASSISTANT.hex` 烧录到 U1 (GD32F470)：

```bash
# J-Link 方式
JLinkExe -device GD32F470ZK -if SWD -speed 4000 -autoconnect 1 -CommanderScript flash.jlink

# 或者使用 OpenOCD
openocd -f interface/jlink.cfg -f target/gd32f4xx.cfg -c "program PLANT_ASSISTANT.hex verify reset exit"
```

## 六、运行方式

### 6.1 固件运行

1. 烧录完成后，复位 U1 或重新上电
2. 系统自动初始化所有子板（扫描 I2C 总线、初始化传感器和执行器）
3. 进入主循环，每 1 秒读取一次传感器、计算评分、执行自动控制

### 6.2 PC 前端控制面板启动

PC 前端是一个基于 Python Flask 的 Web 界面，通过串口与 U1 通信。

```bash
# 进入前端目录
cd plant_assistant/frontend/

# 安装依赖
pip install -r requirements.txt

# 启动前端服务器
python server.py
```

启动后在浏览器打开 **http://localhost:5000** 即可看到控制面板。

### 6.3 前端控制面板功能

| 功能区域 | 说明 |
|----------|------|
| **环境数据** | 实时显示温度、湿度、光照、超声波距离 |
| **评分系统** | 温度分、湿度分、光照分、综合评分 (进度条 + 数字) |
| **植物切换** | 一键切换绿萝/多肉/发财树，自动适配不同养护参数 |
| **窗帘控制** | 手动打开/关闭/设置位置百分比，或切换到自动模式 |
| **风扇控制** | 手动开启/关闭/设置转速百分比，或切换到自动模式 |
| **系统模式** | 全自动模式 vs 手动模式切换 |
| **串口命令** | 直接发送任意串口指令进行调试 |

### 6.4 串口命令协议

PC 前端通过 UART0 (115200 bps) 发送文本命令，U1 接收并响应：

| 命令 | 说明 | 示例响应 |
|------|------|----------|
| `TEMP?` | 查询温度 | `TEMP:25.3 C` |
| `HUMI?` | 查询湿度 | `HUMI:62.1 %` |
| `LIGHT?` | 查询光照 | `LIGHT:1200 lux` |
| `SCORE?` | 查询评分 | `SCORE:T=90 H=85 L=80 Total=86` |
| `STATUS?` | 查询全部状态 | 完整系统状态报告 |
| `PLANT:N` | 切换植物档案 | `PLANT:Switched to 绿萝` |
| `PLANT:NEXT` | 下一个植物 | `PLANT:Switched to 多肉` |
| `CURTAIN:OPEN` | 打开窗帘 | `OK` |
| `CURTAIN:CLOSE` | 关闭窗帘 | `OK` |
| `CURTAIN:POS,N` | 设置窗帘位置N% | `OK` |
| `CURTAIN:AUTO` | 恢复窗帘自动 | `OK` |
| `FAN:ON` | 开启风扇 | `OK` |
| `FAN:OFF` | 关闭风扇 | `OK` |
| `FAN:SPEED,N` | 设置风扇转速N% | `OK` |
| `FAN:AUTO` | 恢复风扇自动 | `OK` |
| `AUTO:ON` | 全自动模式 | `OK` |
| `AUTO:OFF` | 手动模式 | `OK` |
| `HELP` | 帮助列表 | 所有支持的命令 |

## 八、功能说明

### 8.1 自动养护逻辑

系统根据当前选中的**植物档案**自动判断环境是否合适：

| 植物 | 温度范围 | 湿度范围 | 光照范围 |
|------|----------|----------|----------|
| 绿萝 | 18-28°C | 50-80% | 500-15000 lux |
| 多肉 | 15-30°C | 30-60% | 2000-30000 lux |
| 发财树 | 20-32°C | 40-75% | 1000-25000 lux |

**自动控制规则：**
- 光照过强 (超过 shade_high) → 关闭窗帘
- 光照不足 (低于 shade_low) → 打开窗帘
- 湿度过高 (超过 vent_high) → 开启风扇
- 湿度降低 (低于 vent_low) → 关闭风扇
- 蒸发指数超阈值 → WiFi 推送浇水提醒 + ZigBee 广播 + 蜂鸣

### 8.2 评分系统

每个传感器按与目标范围的偏差打分 (0-100)，综合分 = 温度×30% + 湿度×40% + 光照×30%。

| 评分 | 颜色 | 含义 |
|------|------|------|
| 90-100 | 绿色 | 优秀 |
| 70-89 | 黄色 | 良好 |
| 50-69 | 橙色 | 一般 |
| < 50 | 红色 | 差 |

### 8.3 植物切换方式

1. **PC 前端界面** 点击切换按钮
2. **PC 串口命令** `PLANT:1` / `PLANT:2` / `PLANT:3`
3. **NFC 刷卡** 刷对应植物的 NFC 标签
4. **U2 视觉识别** (可选) 摄像头识别植物后自动切换

### 8.4 浇水提醒

系统持续累加蒸发指数 (温度 × 0.3 + 光照 × 0.5 - 湿度 × 0.2)。当蒸发指数超过植物设定的 threshold 时：
1. C3 WiFi 发送 HTTP 浇水提醒
2. C2 ZigBee 广播浇水消息
3. 扬声器蜂鸣提醒
4. 蒸发指数清零，进入冷却期

## 九、常见问题

**Q: U1 子板和母板是什么关系？**
A: 母板(M1)是被动的底座平台，U1 是插入母板的**计算子板**。主控程序运行在 U1 (GD32F470) 上，不是母板上。

**Q: 温湿度是哪个子板？**
A: 温湿度传感器 (SHT35) 集成在 **S2 三合一传感器板**上。S2 同时包含温湿度、光照和 6 轴传感器。不存在独立的 S8 子板。

**Q: PC 前端如何连接？**
A: PC 通过 USB 连接 U1 计算子板，前端 Python 程序通过串口 (COM 口) 与 U1 通信。启动 `python server.py` 后，用浏览器打开 http://localhost:5000。

**Q: 编译需要什么工具链？**
A: 需要 ARM GCC 工具链 (`arm-none-eabi-gcc`)，可以从 [ARM Developer](https://developer.arm.com/tools-and-software/open-source-software/developer-tools/gnu-toolchain) 下载。

**Q: 多块同类型子板如何区分？**
A: 通过子板上的拨码开关调整 I2C 地址偏移。代码中 `get_board_address()` 会扫描多个地址找到实际存在的子板。
