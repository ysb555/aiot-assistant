## 项目名称
小米AIoT智能植物养护助手（软件功能总结）

## 核心功能模块

### 1. 环境监测与评分系统
- 每秒采集：温度、湿度、光照
- 三个维度分别打分（0-100分），然后加权综合：
  - 温度分：适宜区间内100分，超出每单位扣10分（最低0）
  - 湿度分：适宜区间内100分，超出每单位扣2分（最低0）
  - 光照分：适宜区间内100分，超出每2单位扣1分（最低0）
- 综合评分 = 温度分×0.3 + 湿度分×0.4 + 光照分×0.3
- 评分等级（0-100）映射到RGB颜色：
  - 90-100：绿色
  - 70-89：黄色
  - 50-69：橙色
  - 0-49：红色
- 数码管显示综合评分（整数）

### 2. 自动遮阳（回差控制）
- 光照超过高阈值 → 窗帘完全关闭
- 光照低于低阈值 → 窗帘完全打开
- 光照介于高低阈值之间 → 窗帘保持当前位置
- 高低阈值根据当前植物档案动态调整

### 3. 自动通风（回差控制）
- 湿度超过高阈值 → 风扇开启（高速）
- 湿度低于低阈值 → 风扇关闭
- 湿度介于两者之间 → 风扇保持当前状态
- 阈值根据植物档案动态调整

### 4. 浇水提醒（无土壤湿度传感器）
- 每秒计算蒸发增量 = 温度×0.3 + 光照×0.5 - 湿度×0.2
- 累加到蒸发指数（全局变量）
- 当蒸发指数超过预设阈值（默认500）时：
  - 通过WiFi发送HTTP GET请求到手机端（提醒浇水）
  - 蒸发指数清零
  - 进入1小时冷却期，期间不再重复发送

### 5. 植物档案切换
- 预置3种植物（如绿萝、多肉、发财树），每种有独立的：
  - 温度适宜区间（下限、上限）
  - 湿度适宜区间（下限、上限）
  - 光照适宜区间（下限、上限）
  - 遮阳触发高阈值、低阈值
  - 通风触发高阈值、低阈值
- 短按切换按键：循环切换植物编号（1→2→3→1）
- 切换成功时：
  - 数码管第一位显示当前植物编号
  - RGB灯闪烁2次
  - 自动更新上述所有动态阈值

## 主循环结构（轮询，无RTOS）
```
while(1) {
    每10ms循环一次，使用tick计数

    if (tick % 1000 == 0) {   // 每秒执行
        读取温湿度、光照
        计算各指标分和综合分
        更新数码管显示
        更新RGB灯颜色
        执行自动遮阳逻辑（基于当前光照和植物阈值）
        执行自动通风逻辑（基于当前湿度和植物阈值）
        累加蒸发指数
        检查是否触发浇水提醒
    }

    if (tick % 50 == 0) {     // 每50ms执行
        扫描按键
        处理按键（切换植物档案等）
    }

    tick += 10;
    延时10ms
}
```

## 需要的软件接口（供AI实现）

### 传感器与评分
- `读取温度()`
- `读取湿度()`
- `读取光照()`
- `计算综合评分()`
- `更新数码管和RGB灯()`

### 执行器控制
- `设置窗帘位置(0-100%)`
- `设置风扇转速(0-100%)`
- `自动遮阳逻辑()`（含回差，依赖当前植物阈值）
- `自动通风逻辑()`（含回差，依赖当前植物阈值）

### 通信与交互
- `WiFi连接()`
- `发送浇水提醒()`（构造HTTP GET请求）
- `获取当前植物编号()`
- `切换植物档案()`（更新全部动态阈值，闪烁提示）
- `主循环()`

### 蒸发指数
- 全局变量：`蒸发指数累加值`
- `更新蒸发指数()`（每秒调用）
- `检查浇水提醒()`（超阈值则发送并清零，带冷却标志）

## 植物档案数据结构（示例）
每个植物包含：
- 编号
- 温度下限、上限
- 湿度下限、上限
- 光照下限、上限
- 遮阳高阈值、低阈值
- 通风高阈值、低阈值

## 其他
- 需要按键消抖或简单状态机处理短按
- 浇水提醒冷却期可用时间戳或简单计数器实现
- 所有阈值切换后立即生效，无需重启
