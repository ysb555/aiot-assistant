### 语音控制模块使用说明

#### 系统环境要求
- 开发环境：Linux x86_64
- 目标架构：ARM (需交叉编译)

#### 环境配置步骤
1. 配置交叉编译工具链环境变量

   我使用的是这个链接下载的https://huggingface.co/csukuangfj/sherpa-ncnn-toolchains/tree/main
```bash
export PATH=$PATH:/path/to/gcc-arm-8.3-2019.03-x86_64-arm-linux-gnueabihf/bin
```
2. 修改主代码参数：编辑`sherpa-ncnn-beta/sherpa-ncnn/csrc/sherpa-ncnn-alsa.cc`文件

3. 配置网络参数：在`sherpa-ncnn-beta/sherpa-ncnn/csrc/c3.h`文件中设置
   - SSID: 更改为与主机相同的WiFi名称
   - PASSWD: 修改为对应的WiFi密码
   - PORT: 保持默认值(通常无需修改)
   
   **网络连接**：
   
   - 确保模块与主机在同一局域网内
   - 修改IP地址：在`sherpa-ncnn-alsa.cc`文件的第245行
   - 网络通信初始化：代码第328行调用`c3_init(&fd, SSID, PASSWD, (char *)ip.c_str(), PORT)`

#### 编译与部署流程
1. 编译项目(在项目根目录执行)
```bash
./build-arm-linux-gnueabihf.sh
```
2. 上传可执行文件到目标设备
```bash
adb push build-arm-linux-gnueabihf/install/bin/sherpa-ncnn-alsa /userdata/talsa
```
3. 上传语音模型文件

   来源于kaldi，在kaldi官网找一找
```bash
adb push your_smallmodel_path /userdata/smallmodel
```

#### 运行指令

**音频播放功能**：

- 提示音文件需放置在`/userdata/wavs`目录下

- 可通过以下命令上传并解压音频文件：

  ```bash
  adb push wavs.tar /userdata/
  adb shell "cd /userdata && tar xvf wavs.tar"
  ```

- 已知问题：异步播放的音频可能被麦克风采集，当前通过sleep机制缓解

```bash
adb shell
cd /userdata
./talsa ./smallmodel/tokens.txt ./smallmodel/encoder_jit_trace-pnnx.ncnn.param \
./smallmodel/encoder_jit_trace-pnnx.ncnn.bin ./smallmodel/decoder_jit_trace-pnnx.ncnn.param \
./smallmodel/decoder_jit_trace-pnnx.ncnn.bin ./smallmodel/joiner_jit_trace-pnnx.ncnn.param \
./smallmodel/joiner_jit_trace-pnnx.ncnn.bin hw:0,1 4 greedy_search 2> /dev/null
```

#### 注意事项
1. **WiFi连接限制**：
   - 模块当前未实现WiFi连接超时和服务器连接超时处理机制
   - 若WiFi或服务器连接失败，可能导致主进程阻塞甚至崩溃
   - 本实现未考虑离线工作场景

#### **守护进程**：

```bash
chmod 777 talsa_dameon.sh  # 赋予执行权限
nohup ./talsa_dameon.sh 2>&1 >/dev/null &  # 后台启动守护进程
```

#### 调试建议
- 移除`2> /dev/null`可查看详细运行日志
- 使用NetAssist应用(Windows)接收识别结果，也可自行开发简单应用
