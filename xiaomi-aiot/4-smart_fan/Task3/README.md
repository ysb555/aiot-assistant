# 智能风扇命令系统使用指南

## 一、系统简介
本系统通过Zigbee接收命令，实现智能风扇的开关控制、参数设置、状态显示及NFC功能操作。支持6类基础命令，可通过串口或Zigbee终端发送指令。


## 二、基础命令列表（大小写不敏感）
| 命令    | 功能描述                         | 格式示例                   |
| ------- | -------------------------------- | -------------------------- |
| `turn`  | 控制风扇开关（on/off）           | `turn on` / `turn off`     |
| `set`   | 设置温度阈值或工作模式           | `set t1 25` / `set normal` |
| `dis`   | 控制显示屏内容（温度/湿度/关闭） | `dis t` / `dis off`        |
| `check` | 查看温度阈值或风速档位           | `check t2` / `check s1`    |
| `test`  | 测试系统LED灯状态                | `test`                     |
| `nfc`   | 配置NFC标签数据或权限            | `nfc write 10 20 30 40`    |


## 三、命令详解与示例
### 1. 风扇开关：`turn`
- **功能**：控制风扇启停，开启时自动显示温度。  
- **参数**：  
  - `on`：启动风扇（运行标志`run_flag=1`）。  
  - `off`：关闭风扇（运行标志`run_flag=0`）。  
- **示例**：  
  ```bash
  turn on    # 启动风扇
  turn off   # 关闭风扇
  ```

### 2. 参数设置：`set`
- **功能**：调整温度阈值或切换工作模式（普通/开发/NFC读取）。  
- **参数**：  
  - `t1 <值>`：设置低温阈值（范围：>0，默认20℃）。  
  - `t2 <值>`：设置高温阈值（范围：>0，默认38℃）。  
  - `normal`：普通模式（默认）。  
  - `developer`：开发者模式（启用NFC写入）。  
  - `reader`：NFC读取模式。  
- **示例**：  
  ```bash
  set t1 28    # 设置低温阈值为28℃
  set developer # 切换至开发者模式
  ```

### 3. 显示控制：`dis`
- **功能**：切换显示屏显示内容或关闭显示。  
- **参数**：  
  - `t`：显示当前温度（`dis_flag`置位温度位）。  
  - `h`：显示当前湿度（`dis_flag`置位湿度位）。  
  - `off`：关闭显示屏（`dis_flag=0`）。  
- **示例**：  
  ```bash
  dis h    # 显示湿度
  dis off  # 关闭显示
  ```

### 4. 参数查看：`check`
- **功能**：查看已设置的温度阈值或风速档位值。  
- **参数**：  
  - `t1`：查看低温阈值（默认20℃）。  
  - `t2`：查看高温阈值（默认38℃）。  
  - `s1`：查看低速风速（默认40%）。  
  - `s2`：查看高速风速（默认90%）。  
- **示例**：  
  ```bash
  check t2    # 查看高温阈值
  check s1    # 查看低速档位风速
  ```

### 5. 系统测试：`test`
- **功能**：测试LED灯是否正常工作（交替闪烁两次）。  
- **参数**：无  
- **示例**：  
  ```bash
  test    # 执行LED测试
  ```

### 6. NFC控制：`nfc`
- **功能**：向NFC标签写入数据或设置访问权限。  
- **参数**：  
  - `write <v1> <v2> <v3> <v4>`：写入4个字节数据（如风速、阈值等）。  
  - `priv <权限值>`：设置权限（值与`0xe0`按位与，如`60`→`0b00111100 & 0xe0=0b00100000`）。  
- **示例**：  
  ```bash
  nfc write 25 80 22 35    # 写入4个参数值
  nfc priv 60             # 设置权限值
  ```


## 四、使用流程
1. **连接设备**：确保Zigbee模块已组网，设备处于可接收指令状态。  
2. **发送命令**：通过串口工具（如Tera Term）或Zigbee终端发送指令，格式为纯文本（如`turn on`）。  
3. **查看反馈**：  
   - 成功执行：设备返回`OK`。  
   - 失败执行：设备返回`ERROR`，检查参数格式或范围。  


## 五、注意事项
1. **参数格式**：命令与参数用空格分隔，数值参数需为整数（如`set t1 25`）。  
2. **阈值范围**：温度阈值需大于0，风速档位需在0-100之间。  
3. **NFC操作**：执行`nfc`命令前需确保NFC卡片已贴近设备，且处于开发者模式（`set developer`）。  


## 六、常见问题
- **命令无响应**：检查Zigbee网络连接是否正常，设备是否处于终端模式（`zigbee_mode=TERMINAL`）。  
- **显示异常**：确认`dis`命令参数正确（如`dis t`而非`dis temp`）。  
- **NFC写入失败**：确保权限设置正确（`priv`值需与功能匹配），且卡片空间充足。  

如需扩展命令，可参考代码中`cmd_table`结构体添加新指令项。
